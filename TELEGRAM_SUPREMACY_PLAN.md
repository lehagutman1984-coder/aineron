> Создан: 2026-07-02 | Статус: план к реализации

## 1. Позиционирование: где рынок и где мы

### Рынок РФ (июль 2026) — все конкурируют по одной оси

| Сервис | Суть бота | Чего нет |
|---|---|---|
| BotHub | много моделей, YouTube-summary, чтение ссылок | агентность, память, подписки в боте |
| GPTunnel | pay-as-you-go, GPT-ассистенты (пресеты) | проактивность, research, business |
| ChadGPT | широчайший медиа-стек, презентации, голосовой режим | задачи по расписанию, цитаты источников |
| Syntx AI (800k MAU) | face swap, inpainting, аватары | всё агентное |
| GoGPT | инкогнито-режим, custom templates | всё агентное |

**Ось, по которой не конкурирует никто в РФ: автономность и проактивность.**
Глобальные лидеры уже там: ChatGPT Tasks + Agent mode, Perplexity — scheduled tasks прямо
в WhatsApp, Gemini — Scheduled Actions и Daily Brief. В российском Telegram эта ниша пуста.

### Наш фундамент (уже есть, конкуренты догоняют годами)

32 хендлера: чат со стримингом, изображения/видео/стикеры, голос ASR+TTS, файлы,
история, /memory (Persistent Memory + RAG), /persona, /projects (Project Spaces),
/digest, /remind, /poll, inline, группы с орг-биллингом, admin/рассылка, Mini App,
рублёвый биллинг в копейках (BILLING_MIGRATION_PLAN.md), конкурентные цены (PRICING.md).

### Технологическое окно возможностей

Bot API 10.1 (июнь 2026) дал фичи, «заточенные под AI-ботов», которые почти никто ещё
не внедрил: **нативный стриминг** (`sendMessageDraft`), **Rich Messages** (таблицы, код,
thinking-блоки), **Stars-подписки**, **партнёрская программа Telegram**, **Business
Connection** (бот отвечает от имени пользователя), **топики в личке**, **Managed Bots**.
aiogram 3.29 поддерживает всё. Кто внедрит первым — выглядит на поколение впереди.

## S0 — Фундамент: aiogram 3.29 + слой совместимости

**Зачем.** Сейчас `aiogram>=3.17` — этого мало для Bot API 9.3+ фич. Всё ниже зависит от S0.

- `src/requirements.txt`: `aiogram>=3.29,<4.0`. Прогнать бота на staging (webhook-путь,
  FSM, платежи XTR — pre_checkout критичен).
- Новый модуль `src/telegram_bot/capabilities.py`: определение поддержки фич клиентом
  недоступно ботам, поэтому — фиче-флаги env: `TG_NATIVE_STREAMING`, `TG_RICH_MESSAGES`,
  `TG_STARS_SUBSCRIPTIONS`, `TG_BUSINESS`, `TG_AFFILIATE`. Каждая супер-фича включается
  отдельно и откатывается без деплоя кода.
- Fallback-обёртки в `notify.py`: `send_rich_or_markdown()`, `stream_draft_or_edit()` —
  старый путь (edit_message с троттлингом 3.5 с) остаётся резервным.

**Результат:** платформа готова, риск изолирован флагами.

## S1 — Стриминг и Rich-ответы нового поколения

**Зачем.** Ответ бота — ядро продукта, его видит каждый пользователь в каждом сообщении.
Сейчас: edit_message раз в 3.5 сек, Markdown плющится в plain-text, таблицы разваливаются.
После S1 ответы выглядят лучше, чем у любого бота РФ, и не хуже ChatGPT-приложения.

### Фичи

1. **Нативный стриминг** — `sendMessageDraft` (Bot API 9.5): токены LLM появляются
   «как печатаются», без лимитов edit-троттлинга. Заменить механизм в `handlers/chat.py`.
2. **Rich Messages** — `sendRichMessage` / `sendRichMessageDraft` (10.1): конвертер
   Markdown-ответа LLM → rich-блоки: `RichBlockTable` (таблицы!), `RichBlockPreformatted`
   (код с подсветкой), `RichBlockMathematicalExpression` (формулы — киллер для студентов),
   `RichBlockThinking` для reasoning-моделей (o3, R1, GPT-5 thinking) — блок «размышления»
   как в ChatGPT. Новый модуль `src/telegram_bot/rich.py` (md → RichBlock-дерево, юнит-тесты).
3. **Реакции-статусы вместо «Генерирую...»**: `setMessageReaction` — бот ставит реакцию
   «глаз» на сообщение пользователя при приёме, меняет при готовности. Дёшево, живо,
   освобождает от служебных сообщений. `handlers/chat.py`, `images.py`, `video_cmd.py`.
4. **Message effects** при успешной оплате и завершении долгих генераций (`payment.py`,
   `notify.py`).
5. **Кнопка copy_text «Скопировать»** под ответами с кодом/промтами (Bot API 7.11).
6. **Стилизованные кнопки** (9.4): `icon_custom_emoji_id` + `style` — фирменные иконки
   моделей (GPT/Claude/Gemini) на кнопках выбора — соответствует правилу «без эмодзи»,
   иконки брендов вместо текста. Требует Premium на владельце бота — за флагом.

### Файлы
`requirements.txt`, `bot.py`, `notify.py`, `rich.py` (новый), `capabilities.py` (новый),
`handlers/chat.py`, `handlers/images.py`, `handlers/video_cmd.py`, `handlers/payment.py`,
`keyboards.py`.

### Биллинг
Без изменений (та же генерация, другой транспорт).

## S2 — AI-Задачи: проактивный агент по расписанию

**Зачем.** Главное «белое пятно» рынка РФ. ChatGPT Tasks / Perplexity WhatsApp scheduled
tasks / Gemini Scheduled Actions — а в русском Telegram этого нет ни у кого. Мессенджер —
идеальный канал: бот и так умеет писать первым. Это фича ежедневной привычки: пользователь,
получающий утренний AI-бриф, не уходит к конкуренту.

### Фичи

1. **Модель `AITask`** (`src/telegram_bot/models.py`): `user`, `prompt`, `schedule`
   (cron-выражение или once/daily/weekly + время), `use_web_search`, `network`,
   `is_active`, `last_run_at`, `runs_count`, `max_runs`, `created_from` (bot/web).
   Обобщает существующие `Reminder` (напоминание = задача без LLM) и digest
   (дайджест = встроенная задача) — оба мигрируют на AITask, старые команды остаются алиасами.
2. **/task — FSM-конструктор**, но главный путь — **естественный язык**: пользователь пишет
   «каждое утро в 8 присылай новости AI и курс доллара», LLM-парсер (`DeepSeek V3`, дёшево)
   извлекает prompt + расписание + web_search, бот показывает карточку подтверждения.
   Хендлер `handlers/tasks_cmd.py` (новый). Детект интента прямо в чате: фразы
   «каждый день…», «следи за…», «напоминай…» → предложение создать задачу.
3. **Типовые сценарии** (пресеты в /task): утренний бриф по темам, мониторинг новостей
   по ключевому слову, курс валют/крипты, еженедельная сводка расходов на AI,
   повторяющийся промт (пост в канал каждый понедельник).
4. **Исполнение**: Celery-beat каждую минуту `run_due_ai_tasks()` → на задачу
   `execute_ai_task.delay(task_id)`: web-поиск (Tavily) при флаге → LLM → доставка
   в Telegram (rich-сообщение). Списание = обычная цена сообщения модели (PRICING.md),
   идемпотентный reference `aitask:{task_id}:{run_iso}`.
5. **Лимиты**: free — 1 активная задача, платные тарифы — 3/10/30 (Старт/Стандарт+/Про+).
   Дневной cap исполнений, авто-пауза при балансе ниже стоимости запуска (+уведомление
   с кнопкой «Пополнить»).
6. **Управление**: /tasks — список карточками, пауза/запуск/удаление, «прислать сейчас».
   Зеркало на вебе `/account/tasks/` (DRF `api/views/tasks.py`) — задачи общие для веба и бота.

### Файлы
`models.py` (+миграция), `handlers/tasks_cmd.py` (новый), `tasks.py` (execute/run_due),
`setup_periodic_tasks.py`, `api/views/tasks.py` (новый), `api/urls.py`,
`frontend/app/account/tasks/page.tsx` (новый), `keyboards.py`, `handlers/menu.py`.

### Юнит-экономика
Каждый запуск оплачивается по цене модели (1–4 ₽) — фича сама генерирует ежедневную выручку:
1000 пользователей × 1 утренний бриф (DeepSeek + Tavily) = ~1000 ₽/день при себестоимости ~150 ₽.

## S3 — Deep Research с источниками в боте

**Зачем.** У глобальных лидеров Deep Research — базовое ожидание (есть даже в бесплатном
Perplexity WhatsApp). У российских ботов — максимум «веб-поиск одним запросом» без цитат.
Синергия: на вебе Deep Research уже запланирован (MEMORY_SUPREMACY_PLAN.md, спринт 2) —
здесь переиспользуем движок, бот получает фичу почти бесплатно.

### Фичи

1. **/research <вопрос>** — многошаговый пайплайн (`src/aitext/research.py`, общий
   с вебом): декомпозиция вопроса → 5–10 запросов Tavily → чтение источников →
   синтез отчёта с нумерованными цитатами [1][2] и списком URL.
2. **Живой прогресс**: черновик через `sendMessageDraft` — «Изучаю источник 4/9:
   habr.com…» — пользователь видит работу агента в реальном времени (эффект Perplexity).
3. **Доставка**: rich-сообщение (заголовки, таблица источников) + файл-экспорт
   (Markdown-документ через существующий механизм /export; PDF — later).
4. **Цена**: 10 ₽ фикс (себестоимость ~2–3 ₽: 10 Tavily-запросов + 2 LLM-вызова
   DeepSeek/GPT-4o-mini + синтез Sonnet). Подтверждение цены перед запуском.
   Reference `research:{message_id}`.
5. **Квоты Tavily**: 1000/мес бесплатных — при росте включить платный план Tavily
   (учтено в цене 10 ₽) или добавить второй провайдер поиска за флагом.
6. Меню: кнопка «Исследование» в reply-клавиатуре (место есть — сетка 3×3).

### Файлы
`src/aitext/research.py` (новый, общий движок), `handlers/research_cmd.py` (новый),
`tasks.py` (Celery `run_deep_research`), `keyboards.py`, `menu.py`,
`api/views/research.py` (веб-эндпоинт, если веб-спринт ещё не сделан — делаем здесь).

## S4 — Монетизация 2.0: Stars-подписки, партнёрка Telegram, подарки

**Зачем.** Сейчас в боте только разовые покупки XTR; рекурренты — только Robokassa на вебе.
Stars-подписки дают **MRR прямо в Telegram** (двух-тапная оплата), партнёрская программа —
**виральный рост чужими руками** (каналы сами рекламируют бота за комиссию). Ни один
российский AI-бот это не внедрил.

### Фичи

1. **Stars-подписки** (Bot API 8.0): `createInvoiceLink(subscription_period=2592000)` —
   тарифы Старт/Стандарт/Про как ежемесячные подписки в Stars. Продление приходит
   `successful_payment` с `is_recurring=True` → идемпотентное начисление
   (`reference=telegram_payment_charge_id`, механизм уже есть). Тариф-карточки
   в `handlers/payment.py`, синхронизация с `UserSubscription`.
   Курс XTR↔₽ зафиксировать в конфиге, пересматривать при изменении цен Telegram.
2. **Партнёрская программа Telegram** (Bot API 8.1): включить в BotFather процент
   (рекомендация: 10–15%), обработка `TransactionPartnerAffiliateProgram` в статистике.
   Комиссии выплачивает сам Telegram — нулевая поддержка с нашей стороны. Собственная
   реферальная программа остаётся (для веба); в /referral добавить блок «Для каналов».
3. **Подарки за активность**: `sendGift` (умеренно, за Stars с баланса бота) — награда
   за 30-дневный streak или 3+ приведённых друзей. Флаг, лимит бюджета в env.
4. **Paid Media в канале** (7.5): эксклюзивные подборки генераций (Sora 2 Pro ролики)
   в официальном канале за Stars — дополнительный канал выручки и витрина качества.
5. **/balance**: блок «Подписка» (статус, дата продления, отмена —
   `editUserStarSubscription`).

### Файлы
`handlers/payment.py`, `handlers/balance.py`, `handlers/referral.py`, `models.py`
(поля подписки в TelegramUser или отдельная модель), `users/models.py` (связка
UserSubscription ↔ stars-подписка), `analytics.py`, BotFather-настройки (операционно).

### Риски
Комиссия Telegram на Stars (~30% App Store / меньше через web) — цены Stars-тарифов
выставить с учётом комиссии, чтобы маржа сходилась с PRICING.md (Stars-тариф допустимо
делать на ~15–20% дороже рублёвого — стандартная практика).

## S5 — AI-Секретарь (Business Connection) — новый B2B-продукт

**Зачем.** Bot API 7.2/9.0/10.0: пользователь Telegram Business подключает нашего бота
к своему аккаунту, и бот **отвечает его клиентам от его имени**. Это готовый рынок:
самозанятые, магазины, эксперты, агентства — все, кто тонет в личке. В РФ конкурентов
нет; глобально это Operator-класс. Отдельный платный продукт, а не фича тарифа.

### Фичи (поэтапно, за флагом TG_BUSINESS)

1. **Подключение**: обработка `business_connection` update → модель
   `BusinessConnection(user, connection_id, rights, is_enabled)`. Онбординг-FSM:
   что бот будет делать, в каких чатах (все/белый список), тон ответов.
2. **Режим «Черновики»** (безопасный старт): входящее сообщение клиента → AI готовит
   ответ по базе знаний (Persistent Memory + файлы пользователя, RAG уже есть) → шлёт
   владельцу на подтверждение: [Отправить] [Изменить] [Игнор]. Ничего не уходит без
   одобрения.
3. **Режим «Автопилот»**: авто-ответ на типовые вопросы (часы, цены, FAQ) с пометкой
   и мгновенным уведомлением владельцу; стоп-слово для передачи диалога человеку.
4. **Утренняя сводка**: «За ночь 7 сообщений: 3 отвечено автоматически, 2 ждут
   решения, 2 спам» — использует механизм AITask из S2.
5. **Чек-листы клиенту** (`sendChecklist`, 9.1 — только через business connection):
   AI превращает договорённости из диалога в чек-лист прямо в чате с клиентом.
6. **Тариф**: «Бизнес» 990 ₽/мес (до 300 авто-ответов, ~1 ₽/ответ сверх) —
   через `setup_tariffs` + Stars-подписку из S4. Себестоимость ответа ~0,3–0,5 ₽
   (DeepSeek/GPT-4o-mini + RAG) — маржа ×3+.

### Файлы
`models.py` (BusinessConnection, BusinessMessage-лог, +миграция),
`handlers/business.py` (новый: connection, business_message, черновики-FSM),
`bot.py` (allowed_updates += business_*), `tasks.py` (сводка), `keyboards.py`,
`setup_tariffs.py` (тариф «Бизнес»), лендинг-страница `frontend/app/business-bot/` (SEO).

### Риски
Требует Telegram Business у владельца аккаунта (с 10.0 Premium не обязателен для
управления через бота — проверить актуальные условия на момент реализации). Приватность:
храним только метаданные + embeddings базы знаний, не логируем переписку клиентов (кроме
очереди черновиков с TTL) — прописать в политике конфиденциальности.

## S6 — Mini App 2.0: из лаунчера в приложение

**Зачем.** Сейчас /tg/ — одна страница с балансом и ссылками. Mini Apps 2.0 (Bot API 8.0)
позволяет сделать полноценное приложение «без App Store». Плюс виральные механики шаринга.

### Фичи

1. **Fullscreen + addToHomeScreen**: полноэкранный режим, промо «Добавить на главный
   экран» → иконка aineron на телефоне пользователя. Ретеншн-механика уровня нативного
   приложения.
2. **Галерея генераций**: все изображения/видео пользователя (данные `/account/files/`
   уже есть) с `downloadFile` (нативное скачивание) и **`shareMessage`** — поделиться
   генерацией в любой чат одним тапом; в сообщении — кнопка «Создать своё» (deeplink) —
   основная виральная петля.
3. **Полный чат в Mini App**: экран чата с моделями (реюз компонентов веб-чата),
   для сценариев, где нужен богатый UI: сравнение моделей, Canvas-документы (синергия
   с веб-Canvas из MASTER_PLAN Sprint 8).
4. **SecureStorage** для JWT (вместо localStorage), **DeviceStorage** для черновиков.
5. **shareToStory**: сгенерированная картинка → стори пользователя с линком на бота.
6. **Main Mini App** в профиле бота + direct-link (`t.me/aineron_bot/app`).

### Файлы
`frontend/app/tg/**` (переработка: роутер экранов Баланс/Галерея/Чат/Настройки),
`frontend/lib/telegram-webapp.ts` (обёртки 2.0-фич), `api/views/telegram_webapp.py`

## S7 — Топики-проекты в личке + Группы 2.0

**Зачем.** Bot API 9.3–9.4 разрешил форум-топики в личных чатах ботов — это нативные
«папки чатов» прямо в Telegram: топик «Работа», топик «Учёба», топик с отдельной персоной.
Плюс групповая суммаризация — то, что Grok показал как must-have нативного TG-ассистента,
а российские боты игнорируют.

### Фичи

1. **Топики в личке** (за флагом): «Проекты» → бот создаёт топик на проект
   (`createForumTopic`), маппинг topic_id ↔ `Project`/`TelegramChat` — у каждого топика
   свой контекст, персона и модель. Естественная навигация вместо /history-переключений.
2. **/summary в группах**: суммаризация за N часов/сообщений (для групп с орг-биллингом
   и лично для запросившего). Дневная авто-сводка «что обсуждали» (опция админа группы,
   механизм AITask).
3. **AI-квизы** (9.6/10.0: медиа в вариантах, несколько правильных): /quiz <тема> —
   генерация квиза для группы, таблица результатов. Виральная механика в группах.
4. **Групповой /stat для B2B** (из бэклога MASTER_PLAN): расход по участникам за период —
   карточкой в группе + деталь в Mini App.

`handlers/topics.py` (новый), `handlers/group.py` (+summary, quiz, stat),
`models.py` (topic_id маппинг), `tasks.py` (авто-сводки).

## S8 — Managed Bots: персональный AI-агент каждому (moonshot)

**Зачем.** Bot API 9.6 (апрель 2026): manager-бот может **создавать персональные боты**
для пользователей (`getManagedBotToken`). Продукт: «Собери своего AI-бота за 2 минуты» —
пользователь получает СВОЕГО бота (@his_name_bot) с выбранной персоной, базой знаний
(RAG) и моделью. Синергия со Studio (шаблон TMA/бота уже есть) и Personas.
Это новая категория рынка: не «доступ к нейросетям», а «фабрика персональных агентов».

### Эскиз (детализация после S1–S5)

- Биллинг: сообщения гостей бота списываются с владельца (орг-механика уже есть);
  тариф «Агент» — фикс/мес + за сообщения.
- Bot-to-bot (10.0) — later: агенты пользователей взаимодействуют между собой.
- Guest queries (10.0): «попробовать без регистрации» для новых пользователей aineron.

## 5. Общие риски

1. **aiogram 3.29 апгрейд** — регресс webhook/FSM/платежей; митигация: staging-прогон
   polling-режимом (`run_bot`), фиче-флаги S0, откат по `requirements.txt`.
2. **Rich Messages на старых клиентах Telegram** — деградация до обычного текста;
   fallback-обёртка в `notify.py` обязательна (S0).
3. **Юнит-экономика проактивных фич** — задачи исполняются без присутствия пользователя;
   жёсткие лимиты по тарифам + авто-пауза при низком балансе (заложено в S2).
4. **Business-приватность** — только режим «Черновики» по умолчанию, «Автопилот» —
   явное включение; политика конфиденциальности до публичного запуска S5.
5. **Комиссии Stars** — маржу Stars-тарифов пересчитать при изменении курса XTR;
   зафиксировать курс в одном конфиге.
6. **Замороженный live-voice** (MASTER_PLAN §C1) остаётся замороженным: Bot API
   по-прежнему не пускает ботов в voice-чаты (проверено до 10.1); вернуться, если
   Telegram откроет API либо через MTProto-компонент — отдельное решение.
