## 0. Как запустить (обязательно перед тестом)

```bash
git pull
bash deploy.sh
# deploy.sh уже делает: migrate (0047, 0016, 0017), setup_periodic_tasks,
# setup_webhook (регистрирует команды меню + business-апдейты), setup_tariffs
```

Разово после первого деплоя новых фич:

```bash
# U2 — эмбеддинги старых резюме чатов (повторять, пока не скажет «все с эмбеддингами»)
docker-compose exec web python manage.py backfill_summary_embeddings

# (опционально) свести старые дайджесты на движок AI-задач
docker-compose exec web python manage.py migrate_digest_to_aitask --dry-run
docker-compose exec web python manage.py migrate_digest_to_aitask
```

Проверка, что webhook и команды встали:

```bash
docker-compose exec web python manage.py setup_webhook --info
# URL должен быть https://aineron.ru/telegram/webhook/, pending updates ~0, без ошибок
```

Если что-то из «волны 2» (см. §7) выключено — включается флагом в `.env` без правки кода,
затем `docker-compose restart web celery_worker`.

## 1. Общая картина: что теперь есть

**Telegram-бот** — не «ещё один GPT-бот», а проактивный AI-агент:
работает по расписанию, исследует с источниками, выполняет многошаговые задачи,
отвечает клиентам за владельца, ведёт каналы, создаёт персональных ботов.

**Веб-платформа + бот делят одно ядро** — память, база знаний, RAG, агент:
всё, что сделано в проекте, накапливается и усиливает следующий ответ.

Ключевая мысль после U1–U6: **память × Spaces × агенты теперь связаны.**
Факт, извлечённый в проекте, живёт в проекте. Отчёт исследования становится
знанием. Агент видит файлы и умеет предлагать правки. Секретарь отвечает по вашему прайсу.

## 3. Как протестить каждую фичу Telegram (S0–S12)

### S1 — Стриминг и Rich-ответы
- Задайте боту вопрос, требующий таблицы: «сравни GPT-4o, Claude, Gemini таблицей».
- **Ожидаемо:** на ваше сообщение появляется реакция 👀 (принято), ответ «печатается»
  вживую, под ответом с кодом — кнопка «Скопировать код». При `TG_RICH_MESSAGES=1` —
  таблица рендерится как таблица.
- Оплатите что-нибудь / дождитесь готовности видео — придёт «эффект праздника».

### S2 — AI-Задачи (флагман)
- Напишите боту обычным текстом: `/task каждое утро в 8 присылай новости AI и курс доллара`.
- **Ожидаемо:** бот разбирает фразу (DeepSeek-парсер) → карточка подтверждения
  (название, промт, расписание, веб-поиск) → «Создать».
- `/tasks` — карточки с кнопками «Пауза / Сейчас / Удалить». Жмите **«Сейчас»** — отчёт
  придёт сообщением, деньги спишутся по цене модели (идемпотентно).
- Веб-зеркало: залогиньтесь на сайте → **Кабинет → AI-задачи** (`/account/tasks/`).
  Создание/пауза/запуск — те же задачи, что в боте.
- Детект интента: напишите в чате «напоминай мне каждый день пить воду» — бот предложит
  кнопку «Создать AI-задачу».

### S3 — Deep Research
- `/research как изменился рынок LLM в 2026 году`.
- **Ожидаемо:** карточка с ценой (10 ₽) → «Запустить». Живой прогресс «поиск 4/9…»,
  затем отчёт с цитатами `[1][2]` + файл `.md`. При ошибке/таймауте — возврат средств.

### S4 — Монетизация: Stars-подписки, партнёрка
- `/subscribe` → список тарифов как ежемесячных подписок в Stars → оформление в 2 тапа.
- `/balance` показывает блок «Подписка» (дата продления, кнопка «Отменить»).
- Партнёрка: `/referral` содержит блок «Для каналов» (при `TG_AFFILIATE=1`).
- Админ: `/paidpost <stars> <@канал>` (ответом на фото), `/affstats` (Stars-транзакции).

### S5 — AI-Секретарь (нужен `TG_BUSINESS=1`)
- В Telegram: Настройки → Telegram Business → Чат-боты → `@aineron_bot`.
- Напишите боту с другого аккаунта как «клиент» → владельцу приходит карточка
  с готовым черновиком и кнопками **[Отправить] [Изменить] [Игнор]**.
- `/secretary` — режим Черновики/Автопилот, вкл/выкл, **«Подключить базу знаний»** (U4).
- Лендинг для продаж: `https://aineron.ru/business-bot/`.

### S6 — Mini App 2.0
- Откройте Mini App (кнопка в боте / `t.me/aineron_bot/app`).
- **Ожидаемо:** нижняя навигация **Баланс / Галерея / Чат**; на телефоне —
  полноэкранный режим и кнопка «Добавить на главный экран». В Галерее — свои генерации
  с кнопками «Скачать / Поделиться / Стори» (шаринг в любой чат одним тапом).

### S7 — Топики и Группы 2.0
- Личка: `/topics` (нужен `TG_TOPICS=1`) → выбор проекта → бот создаёт топик с
  отдельным контекстом.
- Группа (где бот админ): `/summary 8` (сводка за 8 ч), `/quiz история Рима`
  (нативные квиз-опросы), `/stat` (расход по участникам, для орг-групп).

### S8 — Managed Bots (нужен `TG_MANAGED_BOTS=1`)
- `/mybot` → «Создать бота» → **шаблон персоны** (Поддержка/Юрист/Репетитор/Копирайтер)
  или своя → имя → токен из @BotFather.
- **Ожидаемо:** ваш `@свой_бот` отвечает гостям; сообщения списываются с вашего баланса
  (только личные чаты, дневной лимит `MANAGEDBOT_DAILY_CAP`).

### S9 — Agent Mode
- `/agent сравни цены топ-5 видеокарт и посчитай стоимость фермы из 8 штук`.
- **Ожидаемо:** подтверждение цены (5 ₽) → живые шаги «Ищу в интернете… / Вычисляю…»
  → отчёт. Калькулятор безопасный (не выполняет код).
- **U4-усиление:** если у вас активен проект (`/projects`), у агента появляются
  инструменты базы знаний — см. §5.

### S10 — Голос-в-голос
- Отправьте боту голосовое сообщение.
- **Ожидаемо:** бот распознаёт (ASR), отвечает текстом **и голосовым сообщением** (TTS).
- В `/settings` тумблер «Голосовые ответы» теперь озвучивает вообще каждый ответ.

### S11 — Канал-мейкер
- `/channel` → добавьте бота админом канала с правом «Публикация» → пришлите @канал →
  тема/стиль → расписание.
- **Ожидаемо:** создаётся AI-задача, посты выходят в канал по расписанию со свежими
  фактами. В `/tasks` кнопка «Сейчас» = опубликовать немедленно.

## 4. Unified U1–U6: как протестить сшивку

### U1 — Память проекта и организации
- **Проект:** `/projects` → выберите проект → в его чате упомяните факт о себе
  («я пишу на Rust», «мой проект — платёжный шлюз»). После нескольких сообщений
  `/memory` покажет раздел «Проект». В чате ВНЕ проекта этот факт не всплывёт.
- **Веб:** `/account/memory/` — фильтр скоупа (Все / Глобальные / по проекту).
- **Организация (B2B):** owner/admin добавляет общий факт команды через
  `POST /api/v1/orgs/<id>/memory/` (или UI дашборда, когда появится) —
  факт виден всем членам во всех их чатах. Проверка: два аккаунта одной организации.

### U2 — Total Recall (память всех чатов)
- Проведите пару разных диалогов (например, про биллинг и про деплой), дайте им
  «завершиться» (несколько сообщений — резюме сгенерируется в фоне).
- В НОВОМ чате спросите: **«помнишь, что мы решили про биллинг?»**
- **Ожидаемо:** бот подтягивает суть из того старого чата (блок «Из прошлых разговоров»).
  Триггеры: «помнишь», «мы обсуждали», «в прошлый раз», «напомни».
- Веб-поиск по чатам (`/chats/search`) теперь возвращает и семантические совпадения
  (поле `semantic` в ответе) — находит по смыслу, а не только по словам.
- Если recall не срабатывает на старых чатах — прогоните `backfill_summary_embeddings`.

### U3 — Компаундинг исследований
- **Сохранение в KB:** `/research <вопрос>` в чате активного проекта → под отчётом
  кнопка **«Сохранить в базу знаний проекта»**. После этого файл `research-*.md`
  появляется в проекте (`/projects/<id>` → Файлы) и участвует в RAG.
- **Автосохранение:** включите у проекта `auto_save_research` (через админку/API) —
  каждый research проекта сам ложится в KB.
- **Мониторинг по расписанию:** `/task` → пресет **«Мониторинг темы (research)»** →
  тема. Раз в неделю бот делает полный Deep Research, шлёт отчёт И кладёт его в KB
  проекта — следующий отчёт видит прошлые (знания накапливаются).
- Веб-эндпоинт: `POST /api/v1/research/<id>/save/`.

### U4 — Агент с руками + секретарь с базой знаний
- **Агент × Spaces:** сделайте проект активным (`/projects`), загрузите пару файлов,
  затем `/agent что написано в файле X и предложи туда правку`.
- **Ожидаемо:** в шагах агента появляются «Ищу в базе знаний…», «Читаю файл…»,
  «Готовлю правку файла…». Правка НЕ применяется сразу — создаётся **коммит на
  подтверждение** (`/projects/<id>` → раздел коммитов → Подтвердить/Отклонить).
- **Агент на вебе:** `POST /api/v1/agent/` `{goal, project_id}` → поллинг `GET /api/v1/agent/<id>/`.
- **Секретарь × KB:** `/secretary` → «Подключить базу знаний» → выберите проект с
  прайсом/FAQ. Теперь ответы клиентам опираются на файлы проекта (приоритетный источник).

### U5 — Сайт и RSS как источники знаний
- `/projects/<id>` → вкладка Git/Источники → «Подключить» → тип **Сайт** или **RSS**.
- **Сайт:** укажите URL (напр. `https://docs.example.com`) — краулер обойдёт до 50
  страниц того же домена, странички лягут в базу знаний, ресинк раз в сутки.
- **RSS:** укажите URL ленты — новые записи ежедневно добавляются в KB (компаундинг новостей).
- Проверка: после подключения в списке файлов появятся `web-*.md` / `rss-*.md`,
  статус коннектора «синк OK».

### U6 — Качество и доверие
- **Источники в боте:** в чате проекта задайте вопрос по загруженным файлам —
  под ответом строка «— Источники: file1.md, file2.pdf».
- **Фидбек:** кнопки 👍/👎 под ответом пишутся в аналитику (сырьё для тюнинга).
- **Тост памяти в боте:** когда бот что-то запомнил из диалога — приходит
  «Запомнил: …» (не чаще раза в 6 ч).
- **RAG eval (для разработки):**
  ```bash
  docker-compose exec web python manage.py rag_eval \
    --project <id> --golden src/aitext/eval/golden_example.json --top-k 15
  ```
  Печатает `recall@15` и `MRR`; порог качества recall@15 ≥ 0.85 (запускать при
  изменении retrieval-кода). `golden_example.json` — шаблон, замените своими парами.

## 5. Сквозной сценарий «увидеть всю связку» (5 минут)

1. `/projects` → создайте проект «Мой бизнес», загрузите прайс.txt и FAQ.txt.
2. В чате проекта: «какие у нас цены?» → ответ по файлам + строка «— Источники».
3. `/research тренды в моей нише` → «Сохранить в базу знаний проекта».
4. `/agent проверь, соответствует ли наш прайс рыночным ценам из последнего исследования`
   → агент ищет по KB (и прайс, и сохранённый отчёт), сводит, при необходимости
   предлагает правку прайса коммитом.
5. `/task` → «Мониторинг темы (research)» на ту же нишу → еженедельные отчёты копятся в KB.
6. `/secretary` → «Подключить базу знаний» → этот проект. Теперь секретарь отвечает
   клиентам по актуальному прайсу.
7. Через неделю: «помнишь, что показал мониторинг ниши?» → Total Recall достаёт из истории.

Это и есть отрыв: **память помнит, база знаний накапливается, агент действует,
секретарь продаёт — на одном ядре.**

## 6. Проверка «под капотом» (Django admin + Celery)

Django admin (`/admin/`) — новые модели, где всё видно:

| Раздел | Что проверять |
|---|---|
| Telegram → **AI-задачи** | расписание, `next_run_at`, `kind` (llm/research), `project`, счётчик запусков, `paused_reason` |
| Telegram → **Agent Mode запуски** | `status`, шаги, `result_md`, `project` |
| Telegram → **Stars-подписки** | тариф, `xtr_amount`, дата продления |
| Telegram → **Business-подключения / Черновики** | режим, счётчик ответов, очередь черновиков (TTL 7 дней) |
| Telegram → **Топики / Персональные боты** | маппинг топиков, managed-боты |
| aitext → **Факты о пользователях** | поля `project` / `organization` (скоуп памяти U1) |
| aitext → **Глубокие исследования** | `saved_file` (U3 — отчёт в KB) |
| aitext → **Файлы базы знаний** | `source`: upload/repo/**research/web/rss** |

Celery Beat (периодические задачи — `setup_periodic_tasks`):
`run_due_ai_tasks` (каждую минуту), `send_reminders`, `send_daily_digests`,
`offer_first_ai_task` (день-2 онбординг), `admin_weekly_report` (пн — сводка админам),
`business_daily_summary`, `cleanup_business_drafts`, `cleanup_group_message_logs`,
`poll_connectors` (в т.ч. ресинк сайтов/RSS раз в сутки), `notify_low_balance`.

Логи: `docker-compose logs -f celery_worker web` — теги `[memory]`, `[recall]`,
`[deep_research]`, `[research_to_kb]`, `[sync-web]`, `[sync-rss]`.

Метрики в боте: недельный отчёт админам (`admin_weekly_report`) считает
события по типам — `task_run`, `research`, `agent`, `business_reply`, `subscription`.

## 7. Флаги в `.env` (что включено, что за флагом)

**Волна 1 Telegram — ВКЛЮЧЕНО** (полный fallback на старый путь):
```
TG_NATIVE_STREAMING=1   TG_RICH_MESSAGES=1
TG_STARS_SUBSCRIPTIONS=1 TG_AFFILIATE=1
```
**Волна 2 Telegram — включать после операционной подготовки:**
```
TG_BUSINESS=0        # после теста на своём Telegram Business + политика конфиденциальности
TG_TOPICS=0          # после проверки топиков в личке
TG_GIFTS=0           # когда на балансе бота есть Stars
TG_MANAGED_BOTS=0    # после пилота
TG_STYLED_BUTTONS=0  # нужен Premium владельца бота
```
**Unified U1–U6 — ВКЛЮЧЕНО** (откат любой волны без деплоя):
```
MEMORY_PROJECT_SCOPE=1  MEMORY_ORG_SCOPE=1  RECALL_CHATS=1
RESEARCH_TO_KB=1        AGENT_TOOLS_KB=1
CONNECTOR_WEBSITE=1     CONNECTOR_CRAWL_LIMIT=50
```
**Экономика (копейки / XTR / лимиты):**
```
TG_XTR_RATE_KOPECKS=200   AITASK_DAILY_CAP=30
RESEARCH_PRICE_KOPECKS=1000  AGENT_PRICE_KOPECKS=500
BUSINESS_REPLY_PRICE_KOPECKS=100  BUSINESS_TARIFF_ALLOWANCE=300
MANAGEDBOT_DAILY_CAP=300  TG_GIFTS_MONTHLY_BUDGET_STARS=500
```
После смены любого флага: `docker-compose restart web celery_worker`.

Операционно в @BotFather (не код): включить партнёрскую программу (10–15%),
назначить Main Mini App (`t.me/aineron_bot/app`).

## 8. Быстрый смоук-тест (чек-лист)

- [ ] `/start` → меню; обычный вопрос → живой ответ + реакция 👀
- [ ] `/task ...` → карточка → создать → `/tasks` «Сейчас» → пришёл отчёт
- [ ] `/research ...` → прогресс → отчёт с цитатами + `.md`
- [ ] `/agent ...` (с активным проектом) → шаги с «базой знаний» → отчёт
- [ ] Проект: вопрос по файлу → ответ + «— Источники»
- [ ] `/research` в проекте → «Сохранить в базу знаний» → файл появился в проекте
